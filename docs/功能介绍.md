# CC Console · 功能介绍

> 版本 1.3.1 · 一个 Windows 桌面小工具，**实时监控本机正在运行的所有 Claude Code 实例**：谁在跑、谁闲着、用的什么模型、上下文快满了没有——并支持直接对实例下达指令。

系统托盘常驻，每秒刷新，跟随系统明暗主题。

**适用场景**：同时开了多个 Claude Code（不同项目 / 不同模型），想一眼看清全局；或想远程/批量对某个实例发指令、清空、回溯，而不必逐个切到终端窗口。

## 一、总览面板

界面顶部的 Hero 区域，一秒一刷，给出全局快照：

| 指标 | 含义 |
|---|---|
| 🟢 在线 N | 当前运行中的 Claude Code 实例总数 |
| 🔴 忙碌 N | 处于 busy 状态（正在生成回复）的实例数 |
| 🟢 空闲 N | 处于 idle 状态、可接收输入的实例数 |
| 📦 X.Xk tokens | 所有在线实例的累计 Context tokens 合计 |
| 🌓 残留 N | 进程已退出、但 session 文件尚未清理的「幽灵」记录数 |
| ⏱ HH:MM:SS | 实时时钟，同时作为「页面在刷新」的活心跳 |

无实例运行时显示「🌙 当前无实例运行」，主体区显示空状态提示卡片。

## 二、实例卡片

每个被识别的 Claude Code 实例渲染为一张卡片，分左右两部分：**左侧信息区** + **右侧操作区**。

### 信息区字段

| 字段 | 说明 |
|---|---|
| 🔴/🟢/⚪ 状态 | 忙碌 / 空闲 / 未知。忙碌时卡片带**脉冲呼吸动画**，老远就能看到哪个在干活 |
| 工作目录 | 显示路径末两级（如 `...\project\app`），鼠标悬停看完整路径 |
| PID | 系统进程号 |
| 模型 | 最后一次回复所用的模型（如 `claude-sonnet-4-6`、`glm-5.1`）；新会话显示「（新）」 |
| 运行时长 | 从进程启动算起的存活时间，自适应为 秒 / 分钟 / 小时 / 天 |
| 💬 对话主题 | AI 自动生成的会话标题；新会话显示「（新会话·无消息）」 |
| 上下文用量条 | 22 格 Unicode 进度条 + 百分比 + 已用/上限（如 `74%  148k/200k`） |
| ↑ 本轮输出 | 最近一轮回复的输出 tokens |
| 📦 累计 tokens | 整个会话累计的 input / output / cache tokens 明细（仅在有数据时显示） |

### 上下文用量条的颜色分级

| 占用 | 颜色 | 含义 |
|---|---|---|
| < 50% | 默认色 | 充裕 |
| 50% – 80% | 中段色（mid） | 开始吃紧 |
| ≥ 80% | 警示色（high） | 接近上限，建议清空或开新会话 |

> 残留（已退出）实例以暗淡样式排在列表中，便于确认清理情况。

## 三、排序

列表上方提供三个排序维度，点击切换、再点翻转升降序：

| 维度 | 排序依据 |
|---|---|
| **最后活动** | 默认维度。先按「忙碌 → 空闲 → 残留」分组，组内按最近活动时间排序——**最忙的永远在最上面** |
| **建立时间** | 实例启动先后 |
| **Context 用量** | 上下文占用从高到低（或反之），快速找出快爆的会话 |

## 四、实例操作（清空 / 对话 / 回溯 / 窗口）

卡片右侧四个按钮，通过 Win32 控制台输入注入**直接驱动目标实例**，无需切到它的终端窗口：

| 按钮 | 动作 | 说明 |
|---|---|---|
| **清空** | 发送 `/clear` | 清空该实例的当前对话上下文。点击需二次确认 |
| **对话** | 发送自定义文本 | 弹出输入框，写入消息后发送到该实例。**Ctrl+Enter** 发送，多行自动折叠为空格 |
| **回溯** | 发送 `ESC` ×2 | 回溯/中断当前操作 |
| **窗口** | 置前终端窗口 | 把该实例对应的终端窗口拉到最前 |

每次操作完成后，底部 footer 会淡入一条反馈（如「✓ 已向 PID 12345 发送 /clear」），停留数秒后淡出。

> 实现底层是 `AttachConsole` + `WriteConsoleInput`，用 `ConsoleInput` 接口隔离平台差异，因此这部分目前仅 Windows 可用。

## 五、系统托盘与单实例

- **常驻托盘**：关闭主窗口默认**隐藏到托盘继续运行**（可在设置中改为直接退出），只有托盘菜单「退出」才真正结束程序。
- **托盘 Tooltip**：每 2 秒更新，鼠标悬停托盘图标即可看到「在线 N（🔴 忙碌 X · 🟢 空闲 Y）」概要，有残留时额外显示。
- **托盘菜单**：右键「显示窗口 / 退出」。
- **单实例**：第二次启动程序不会开第二个，而是**激活已有窗口**并置前。

## 六、设置面板

点顶部 ⚙ 齿钮打开，左侧三栏导航：

### 常规设置
- **关闭按钮直接退出**：开启后，点窗口关闭按钮直接退出程序，而非最小化到托盘。

### 系统设置
- **开机启动**：开启后随系统启动自动运行。

### 关于
- 应用版本号（当前 1.1.0）
- 跳转 GitHub 仓库

设置即时保存（`~/.cc-console.json`），开机启动同步写入系统启动项。

## 七、主题与动画

- **跟随系统主题**：自动检测 Windows 明/暗模式，切换 Notion 风格的 light / dark 调色板，全程由 CSS 变量驱动，无闪烁。
- **零开销动画**：忙碌脉冲呼吸、footer 消息淡出均由纯 CSS `@keyframes` 实现，不占用 Go 端刷新逻辑。

## 八、命令行模式

不启动 GUI，纯终端打印一次实例表格后退出，适合脚本 / 日志 / 定时采集：

```bash
cc-console.exe --list
```

输出示例：

```

22912    ○ 空闲      glm-5.1        13% 27k/200k      55     Update Claude Code version   C:\Users\Administrator
```

## 九、数据来源与实例识别

### 数据来源

| 信息 | 来源 |
|---|---|
| 状态 / cwd / sessionId / 启动时间 | `~/.claude/sessions/<pid>.json` |
| 模型 / tokens / 主题 | `~/.claude/projects/<编码后的cwd>/<sessionId>.jsonl` |
| 模型上下文上限 | 内置「模型 → 上限」表（60+ 条目）+ `~/.cc-console.json` 的 `modelLimits` 覆盖 |
| Claude Code 模型设置 | `~/.claude/settings.json`（解析 `[xxx]` 标注的环境变量） |

> 大 JSONL 对话文件按 **mtime 缓存**，未变化则不重读，保证每秒刷新不卡。

### 它如何识别「真正的 Claude Code」

本机 `claude.exe` 被**两个产品**共用：

- **Claude Code CLI**（监控对象）：会写 `~/.claude/sessions/<pid>.json`。
- **Claude 桌面版**（Electron）：一堆 `--type=gpu/renderer/utility` 子进程，**不写 session 文件**。

判定标准：**一个 `claude.exe` 只有在「存在对应 session 文件，且进程启动时间与会话 `startedAt` 一致」时才计为实例**（容差 15s，防 PID 复用）。这样能同时：

1. 排除 Claude 桌面版（无 session 文件）；
2. 排除 Claude Code **启动瞬间派生的短命辅助子进程**——否则界面会「启动时冒出好几行、一两秒后消失」。

### 自定义模型上下文上限

Claude Code 数据里的 `contextWindow` 为 0，不可靠。百分比依赖内置上限表估算。若你的模型上限与默认不同，可在 `~/.cc-console.json` 覆盖：

```json
{ "modelLimits": { "glm-5.1": 256000 } }
```

## 十、平台支持

| 平台 | 状态 |
|---|---|
| **Windows** | 完整支持（GUI + 实例操作 + 主题检测） |
| macOS / Linux | 框架已就绪，平台特定逻辑（进程探测 / 输入注入 / 主题检测）暂为存根 |

平台特定代码由 Go `//go:build` build tag 隔离，跨平台扩展时只需补齐对应平台的存根实现。
